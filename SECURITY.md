# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in ooke, please report it responsibly.

**Do not open a public issue.** Instead, email the maintainer directly or use GitHub's private vulnerability reporting feature.

When reporting, please include:

- A description of the vulnerability.
- Steps to reproduce.
- The potential impact.
- Any suggested fix, if you have one.

## Response

You can expect an initial response within 72 hours. We will work with you to understand the issue and coordinate a fix before any public disclosure.

## Scope

This policy covers the ooke framework code in this repository. Issues in the toke compiler or standard library should be reported to the [toke repository](https://github.com/karwalski/toke).
